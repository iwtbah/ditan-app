# 原型迁移到正式前端底座的实施任务提示词

- 记录日期：2026-04-14
- 归档范围：frontend
- 主题标签：migration, prototype, implementation, home-page, architecture, reusable-components
- 来源说明：用户要求将原型页面与交互逐步迁移到正式前端工程底座，并按阶段实施

## 原始提示词

你的任务是把原型中的页面与交互，迁移到我的正式前端底座中，还原原型设计，使其符合工程化规范、可维护、可扩展。

迁移目标：
- 保留原型中的页面布局、视觉信息层级、核心交互逻辑，禁止自己创造原型中没有的样式
- 适配到 ditan-app/ 现有技术栈和目录结构
- 提取可复用组件，而不是把每个页面都写成巨型文件
- 页面路由、状态管理、请求层、类型定义、UI 组件分层都要合理
- 最终结果必须可以本地运行

执行要求：
1. 先分析 prototype/ 和 ditan-app/ 的目录结构、依赖、页面组织方式
2. 制定迁移计划，可以分阶段进行，说明：
- 哪些页面可以直接迁移
- 哪些代码只能参考样式，不能直接复用
- 哪些部分需要重构成通用组件
- 路由、状态、mock 数据、接口类型应如何落位
3. 按计划开始实施，不要一次性大爆改，优先完成最核心链路
4. 每完成一个阶段，确保项目可运行，不要引入明显坏掉的中间状态
5. 尽量复用底座中的已有能力，不重复造轮子

工程约束：
- 使用 TypeScript 严格类型
- 使用函数组件与 hooks
- 样式优先用 Tailwind CSS
- 全局状态用 Zustand
- 服务端数据获取预留 TanStack Query 接入方式
- 路由统一放到 router 模块管理
- 公共 UI 组件放到 src/components
- 业务组件按页面域拆分到 src/features
- API 类型、DTO、mock 数据分开管理
- 不要把原型中的临时命名、脏结构、硬编码路径原样搬进正式工程

重点页面优先级：
1. 首页 / 发现页
2. 商铺详情页
3. 探店笔记详情页
4. 评论/推荐菜模块
5. 个人页或其他次级页面
