# 交互还原、视觉质感还原、布局修正与推荐流重构任务提示词

- 记录日期：2026-04-14
- 归档范围：frontend
- 主题标签：interaction, visual-refinement, tab-bar, glassmorphism, masonry, feed-refactor
- 来源说明：用户要求在现有迁移基础上，对交互层、视觉层、全局布局和推荐流做专项整改

## 原始提示词

当前“迪探”前端页面迁移已基本完成，静态还原度约为 80%-90%。现在进入第二阶段：交互还原、视觉质感还原、布局修正、推荐流重构。

请不要重新做页面迁移，而是在现有代码基础上，针对以下问题做专项整改：

当前问题：
1. 动效与交互设计没有还原
2. 底部 Tab Bar 没有固定在视口底部
3. 设计稿中的玻璃浮空效果没有还原
4. 推荐流 cards 不是瀑布流渲染，导致页面过长、信息密度不合理

你的任务目标：
- 在不破坏现有页面结构的前提下，增强交互层与视觉层还原度
- 优先修复全局布局问题，再优化局部页面体验
- 保持代码可维护，不要为了视觉效果堆砌杂乱样式
- 保证移动端视图逻辑正确

请按以下顺序执行：

第一步：先审查现有实现
输出：
- 当前哪些页面缺少关键动效
- 当前 Tab Bar 为什么没有固定
- 当前视觉样式距离“玻璃浮空”差在哪
- 当前推荐流为什么不是瀑布流，问题出在布局还是数据渲染方式

第二步：制定整改方案
分别给出：
- 动效方案
- Tab Bar 固定方案
- 玻璃浮空视觉方案
- 瀑布流实现方案
说明每项改动涉及哪些文件、是否需要新增公共组件或 hooks

第三步：开始实施，分批修改
优先级：
1. 全局固定底部 Tab Bar
2. 玻璃浮空视觉样式体系
3. 推荐流瀑布布局
4. 页面动效与交互补齐

工程要求：
- 技术栈保持 React + TypeScript + Tailwind CSS
- 动效优先考虑 Framer Motion，如果项目已接入则直接使用；如果未接入，先评估是否值得引入
- Tab Bar 必须是固定底部且安全区处理合理
- 玻璃浮空效果不能只靠半透明背景，要同时处理 blur、边框、高光、阴影、层级
- 瀑布流不能简单用两列假拆分，要选择更合理、可维护的实现方式
- 页面高度与滚动体验要符合移动端产品习惯
- 保持组件拆分清晰，不要把全部逻辑塞进页面文件

输出要求：
每一轮修改后，输出：
- 修改了哪些文件
- 每个修改解决了什么问题
- 当前还遗留什么问题
- 下一步建议

如果你发现当前某些实现路径不合理，可以重构，但不要大面积推翻已完成迁移。
