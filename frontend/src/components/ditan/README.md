# Ditan 共享组件边界

`src/components/ditan` 当前不是通用设计系统层，也不是 feature 私有组件层。

它的定位是：

- 当前产品域的共享展示组件
- 可被多个 feature 和组件展示页复用
- 保持展示层职责，不吸收服务端请求和复杂业务状态

适合放入这里的组件：

- 多个页面都会使用的内容卡片
- 当前产品域共享的基础交互部件
- 当前产品域共享的结构性展示组件

不适合放入这里的内容：

- 只服务于单一业务域的私有组件
- 只服务于布局壳的导航或路由承接组件
- 只服务于组件展示页的演示控件
- 请求逻辑、Query hooks、mutation 逻辑
- 与某个页面强绑定的业务编排

导入约定：

- 统一从 `@/components/ditan` barrel 导入
- 不新增深路径导入到 `content-cards.tsx`、`interactive.tsx`、`structure.tsx`

后续演进：

- 若将来出现跨产品域复用，再考虑拆分到更中性的共享组件层
- 在此之前，优先保持当前目录稳定，不做为“更优雅”而进行的大搬家
