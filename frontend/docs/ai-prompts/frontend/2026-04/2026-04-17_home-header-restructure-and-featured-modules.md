# 首页顶部重设计与首屏引导优化

- 日期：2026-04-17
- 来源范围：首页
- 标签：home、header、featured、ui、最小改动

## 原始提示词

本次任务：请重新设计首页上方结构，优化当前首页顶部层级混乱、占空间过多的问题。

当前页面问题
当前首页顶部结构大致为：
- 城市
- 搜索框
- 探店日记 / 店铺切换
- 分类 Tabs
- 内容流

存在的问题：
1. 顶部层级较多，占用首屏空间
2. 搜索框单独占一行，压缩了内容展示空间
3. “探店日记 / 店铺”切换位置偏下，不够核心
4. Header 主层级和内容切换层级不清晰
5. 不同内容模式下首屏引导模块不够明确

本次重设计目标
1. 将搜索入口压缩到右上角，不再保留整行搜索框
2. 将“探店日记 / 店铺”提升到 Header 中作为一级主切换
3. 分类 Tabs 保留在 Header 下方，作为二级切换
4. 让“探店日记”和“店铺”两种模式拥有不同的首屏推荐模块
5. 节省首屏空间，提升结构清晰度和视觉精致度
6. 保持移动端体验优先，不做超大重构

## 上下文说明

- 首页只做表现层和结构层优化，不改业务定义和其他页面。
- 搜索入口需要压缩成右上角轻量入口。
- 一级主切换与二级分类都需要轻量动效，不做重胶囊样式。
- 首屏引导模块允许做静态占位或前端结构预留，不要求新增后端支持。
- 后续细化要求：
  - 好评榜、避雷榜去掉底部说明小字，提升设计感和吸引力。
  - 今日精选、年度佳作缩成更紧凑、等宽的双卡布局。
  - 首页二级分类去掉下划线，可加入轻量线性图表装饰。
  - 主切换“探店日记 / 店铺”在下滑一定距离后自动弱化隐藏。
  - 修正为：首页下滑时自动隐藏的是二级分类，不是主切换；二级分类选中动效恢复到之前的下划线滑动。
  - 额外修复：从日记详情返回时，TabBar 出现瞬间的透明/毛玻璃闪烁问题。
  - 最新细化：
    - 二级分类激活态不要下划线。
    - 二级分类的收起/展开不要在拖动中发生，改为滚动停住后再切状态。
    - 修复首页首屏推荐卡在模式切换后出现的抖动。
  - 最新调整：
    - 二级分类的自动收起/展开不再依赖触摸手势。
    - 改为基于滚动方向和阈值的自动折叠/自动展开，要求动画丝滑，且不破坏上下滑动平滑感。
  - 精选模块优化：
    - “今日精选 / 年度佳作”从普通信息卡升级为内容推荐入口卡片。
    - 使用横向滑动的大图卡结构，强化背景、遮罩、标题和内容感。
    - 保持与下方探店笔记 Feed 卡片一致的圆角、阴影和间距体系。
  - 精选模块减法优化：
    - 每张精选卡只保留标签和主标题，删除推荐入口、摘要、预览和小图装饰。
    - 精选卡继续保留背景图与轻遮罩，但整体更干净、更像内容入口。
