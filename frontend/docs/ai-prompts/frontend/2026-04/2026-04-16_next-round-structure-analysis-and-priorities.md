# 下一轮工程结构分析与优先级评估

- 日期：2026-04-16
- 来源范围：frontend 工程结构优化协作
- 主题标签：`project-analysis` `structure-optimization` `priority` `risk-control`

## 原始提示词

现在进行项目现状分析，评估下项目工程结构，进行下一轮优化建议分级，提出优化下项，最终目标：
在“页面表现 100% 不变”的前提下，让该项目的前端工程结构更清晰、更稳定、更容易继续开发和维护。硬性要求（最高优先级，绝对不能违反）：
1. 页面 UI 不能发生变化
2. 页面布局不能发生变化
3. 页面交互不能发生变化
4. 页面文案不能发生变化
5. 页面视觉层次不能发生变化
6. 页面动效表现不能发生变化

## 补充上下文

本轮关注的是：

- 扫描当前目录和关键入口
- 评估 pages / features / layouts / api / stores / types 的职责闭环
- 明确下一轮优化建议分级
- 只提出低风险、可回退、不会影响页面表现的优化项
