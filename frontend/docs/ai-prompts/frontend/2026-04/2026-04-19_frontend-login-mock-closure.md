## 原始提示词

你现在是资深前端工程师 + 产品工程师，负责为「迪探」项目新增一套前端登录闭环（当前为假登录流程）。

本次任务目标：

- 实现手机号验证码登录页 UI
- 实现获取验证码交互和倒计时
- 实现验证码假校验
- 实现登录状态持久化
- 实现受限页面路由拦截
- 实现登录成功后回跳来源页面
- 预留第三方登录按钮占位

登录规则：

- 游客默认可进入 App
- 未登录访问“迪探 / 新建日记 / 关注 / 我的”时，跳转登录页并记录来源路径
- 当前阶段不接真实后端、短信服务和第三方登录 SDK
- 手机号格式基本合法且验证码非空即可登录成功

输出要求：

- 说明实现方案
- 列出变更清单
- 解释假登录规则
- 说明风险点
- 给出最多 3 条下一步建议

## 上下文说明

- 保持最小改动，不扩大拦截范围。
- 页面层保持轻量，业务逻辑继续下沉到 feature。

- 后续需要能平滑替换为真实短信验证码和第三方登录。
