# 前端工程结构优化轮次提示词

- 日期：2026-04-15
- 来源范围：Codex 对话归档
- 主题标签：前端工程结构、目录分层、路由组织、状态管理、死代码清理、低风险优化

## 原始提示词

你现在是“前端工程结构优化代理”，请针对以下项目做一轮“仅限工程结构层面”的优化分析与实施建议。

项目路径：
/Users/zhaowenzhuo/Workspace/ditan-app/frontend

当前背景：
- 该项目页面迁移工作目前已完成
- 当前目标不是改业务功能，而是优化前端工程结构
- 本轮重点关注：目录、路由接入、状态管理、组件抽取、公共方法抽离、资源整理、死代码清理
- 如果你判断当前阶段某些更深层的结构优化不适合做，可以明确说明“暂不建议优化”及原因
- 如果目前页面实现方式与理想工程结构差异较大，也需要明确指出优化点与原因

硬性要求（最高优先级，绝对不能违反）：
1. 页面 UI 不能发生变化
2. 页面布局不能发生变化
3. 页面交互不能发生变化
4. 页面文案不能发生变化
5. 页面视觉层次不能发生变化
6. 页面动效表现不能发生变化
7. 禁止为了“结构更优雅”而引入任何会影响页面表现的改动
8. 禁止擅自重写页面
9. 禁止大规模替换技术栈或升级框架方案
10. 所有优化必须遵循“最小改动、低风险、可回退”的原则

你的工作方式要求：
1. 先完整扫描项目目录结构
2. 识别当前工程的页面层、业务层、公共层、路由层、状态层、资源层分布
3. 找出以下问题：
   - 目录职责是否混乱
   - 页面级组件与复用组件是否耦合
   - 路由定义是否清晰，是否存在页面与路由强耦合
   - Zustand/Context/其他状态是否放置合理
   - hooks、utils、types、constants 是否分散或重复
   - 静态资源是否存在散落、重复、命名混乱
   - 是否存在未使用组件、未使用页面、未使用方法、未使用资源、废弃代码
   - 是否存在“原型式代码”残留，没有沉淀为正式工程结构
4. 但注意：不是发现问题就一定要改。请你先判断“当前阶段值不值得改、风险大不大、收益高不高”
5. 所有建议都必须围绕“保持页面表现完全不变”展开

执行原则：
- 优先做低风险、高收益优化
- 优先做不影响页面渲染结果的整理类工作
- 优先做清晰分层，而不是抽象过度
- 优先局部收敛，不做激进重构
- 如果某项优化可能牵一发动全身，请先标记为“暂缓”
- 若发现当前实现与理想结构差异很大，不要直接大改，而是分阶段给出建议

你必须按以下顺序输出结果：

第一部分：项目现状分析
- 输出当前目录结构的总体判断
- 说明当前工程更像“原型迁移后的页面集合”还是“已初步成型的正式前端工程”
- 说明哪些区域结构较合理，哪些区域存在明显问题

第二部分：优化建议分级
按 P0 / P1 / P2 分级输出：
- P0：可以立刻做、低风险、收益高、且不影响页面表现的优化
- P1：建议做，但需要控制范围、分步处理的优化
- P2：当前阶段不建议做，或应放到后续迭代再做的优化

第三部分：每个优化项的详细说明
每项必须包含：
- 优化点
- 现状问题
- 为什么值得做
- 风险等级（低/中/高）
- 是否会影响页面表现（必须明确说明“不会”或说明风险）
- 建议改法
- 是否建议本轮执行

第四部分：本轮建议执行清单
输出一个“只做最小必要优化”的清单，要求：
- 数量控制在合理范围
- 以结构整理为主
- 不包含任何 UI 改造
- 不包含任何交互变更
- 不包含任何视觉调整

第五部分：实施约束
明确列出你在执行过程中必须遵守的规则：
- 不改页面效果
- 不改文案
- 不改动效
- 不做无必要抽象
- 一次只做一小类事情
- 每步完成后给出变更说明、影响范围、风险点、回退建议

第六部分：如果你认为当前阶段“不适合做更深层优化”
请明确说明：
- 哪些更深层优化暂时不要做
- 为什么现在不适合
- 后续到什么阶段再做更合适

额外要求：
- 如果你要实际改动代码，必须先输出“本轮计划改动项”
- 改动时要优先保持目录稳定，不做大搬家
- 对于组件抽取，只有在复用价值明确且不会破坏页面语义时才允许抽取
- 对于状态管理，只有在状态归属明显错误或重复时才允许调整
- 对于路由，优先优化组织方式，不轻易改页面挂载关系
- 对于死代码清理，要确认未被引用后再删
- 对于资源整理，不要修改会影响页面展示的资源引用结果

最终目标：
在“页面表现 100% 不变”的前提下，让该项目的前端工程结构更清晰、更稳定、更容易继续开发和维护。

## 补充说明

- 本轮归档的重点是“仅限工程结构层面”的优化分析与低风险实施建议。
- 所有改动必须满足“最小改动、低风险、可回退”，不允许为结构整洁牺牲现有页面表现。
