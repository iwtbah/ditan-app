# 搜索页功能评审与重设计

- 日期：2026-04-17
- 来源范围：搜索页
- 标签：search、ui、交互、tabbar、最小改动

## 原始提示词

请对“搜索页”进行以下工作：
1. 功能评审
2. 功能优化
3. UI 重新设计
4. 隐藏底部 TabBar
5. 保持与项目整体风格一致

当前页面问题：
- 当前搜索页只有：
  - 返回按钮
  - 搜索输入框
  - 搜索按钮
  - 历史搜索标签

存在的问题：
- 页面信息密度太低
- 搜索页定位过弱，像临时输入页，不像完整搜索页
- 缺少热门搜索、推荐搜索、联想建议等核心模块
- 搜索对象虽然是“日记或店铺”，但页面没有明确搜索结构
- 页面留白过多，缺少内容组织
- 底部 TabBar 干扰搜索场景，不应显示

本次改造目标：
请将搜索页设计成一个完整、清晰、移动端友好的搜索页面。

必须实现的目标：
1. 隐藏底部 TabBar
2. 优化顶部搜索区域
3. 重构默认状态的页面内容
4. 为“输入中状态”和“结果态”预留合理结构
5. 保持最小侵入式改动，不影响其他页面
6. 不新增复杂业务逻辑，只做当前搜索体验所需的结构优化

## 上下文说明

- 搜索页需要按默认态、输入中、结果态三种状态组织结构。
- 可以做前端结构预留，但不能伪造依赖后端的复杂搜索能力。
- 必须隐藏搜索页底部 TabBar，且不能影响其他页面的 TabBar。
- 后续调整要求：搜索页默认态移除“推荐搜店铺 / 推荐搜日记 / 最近浏览”，其中“最近浏览”迁移到我的页承载。
