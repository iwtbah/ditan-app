# Feed 底部拖拽回弹交互

- 日期：2026-04-16
- 来源范围：Codex 对话，迪探前端列表交互优化
- 标签：feed、列表交互、回弹、移动端、最小改动

## 原始提示词

```text
主页 ，关注页，我的页，店铺列表页，这种卡片feed流，还是列表流，都需要做一个交互设计，就是拖拽效果，再提示没有更多动态了，松开会回弹，动画要求丝滑

继续优化，第一，越拉，“没有更多动态了”的结束态样式就越明显，到底部了不拉则不显示
第二；我的页里的结束态样式没有像首页和关注页一样，看起来像在卡片里面
第三：动效预览上不够观感不够丝滑
第四；所有下拉能到底的，都用这个交互设计，能抽象化抽取到组件复用吗，结束态样式同样，如果当前不适合这样做，也可以不做
```

## 上下文说明

- 目标是给首页、关注页、我的页，以及店铺列表类页面补充统一的移动端底部拖拽回弹交互，并让结束态随拖拽强度渐显。
- 约束是最小改动，不重做 UI，不改页面信息结构，只增强现有滚动容器的交互反馈。
- 结束态需要在静止到底时隐藏，仅在继续上拉时出现；“我的”页结束态不能再嵌进卡片内容区。
- 结束态文案继续沿用“没有更多动态了”，并保持现有卡片流与列表流结构不变。
